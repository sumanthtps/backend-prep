## **1. Advanced Assertions in JUnit 5**

JUnit 5 provides **enhanced assertion methods** for more expressive and flexible
validation.

### **1ï¸âƒ£ Using `assertAll()` â€“ Grouped Assertions**

ğŸ“Œ **Executes multiple assertions together** and reports all failures at once.

```java

    assertAll("User Properties",
```

âœ… **Reports all failures instead of stopping at the first one**.

## **2. Parameterized Tests â€“ Running Tests with Multiple Inputs**

JUnit 5 allows running **the same test multiple times** with different input
values.

### **1ï¸âƒ£ `@ValueSource` â€“ Simple Input Values**

ğŸ“Œ **Runs the test multiple times with different inputs**.

```java

    assertTrue(number % 2 == 0 || number % 2 == 1);
}
```

âœ… **Replaces multiple similar test methods**.

---

### **2ï¸âƒ£ `@CsvSource` â€“ Multiple Arguments for a Test**

ğŸ“Œ **Provides multiple inputs as CSV (comma-separated values)**.

```java

    "Hello, 5",
    "JUnit, 5",
    "Test, 4"
})

    assertEquals(expectedLength, input.length());
}
```

âœ… **Allows passing multiple parameters dynamically**.

---

### **3ï¸âƒ£ `@MethodSource` â€“ External Data Provider**

ğŸ“Œ **Fetches test data from a separate method**.

```java

    return Stream.of(
        Arguments.of(2, true),
        Arguments.of(3, false),
        Arguments.of(4, true)
    );

@MethodSource("provideNumbers")

    assertEquals(expected, number % 2 == 0);
}
```

âœ… **Supports complex data types and multiple arguments**.

## **3. Controlling Test Execution Order**

JUnit **does NOT guarantee test execution order by default**, but you can
control it.

### **1ï¸âƒ£ `@TestMethodOrder(OrderAnnotation.class)`**

ğŸ“Œ **Explicitly sets the test execution order**.

```java

    @Test
    @Order(1)
    void testFirst() {
        System.out.println("First test");
    }

    @Test
    @Order(2)
    void testSecond() {
        System.out.println("Second test");
    }
}
```

âœ… **Ensures tests run in a specific order when required**.

## **7. Mocking Dependencies with Mockito**

ğŸ“Œ **Mockito helps mock dependencies in unit tests**.

```java

    @Mock
    UserRepository userRepository;

    @InjectMocks
    UserService userService;

    @Test
    void testFindUserById() {
        when(userRepository.findById(1)).thenReturn(Optional.of(new User("Alice")));

        User user = userService.getUserById(1);
        assertEquals("Alice", user.getName());
    }
}
```

âœ… **Prevents using actual database/network calls in unit tests**.
