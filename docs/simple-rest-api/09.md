## ğŸ”¹ **Step 2: Update `UserDaoService` (Service Layer)**

### ğŸ“Œ **Code:**

```java
@Component
public class UserDaoService {

    private static List<User> users = new ArrayList<>();
    private static int userCount = 3;

    static {
        users.add(new User(1, "Adam", LocalDate.now().minusYears(30)));
        users.add(new User(2, "Eve", LocalDate.now().minusYears(25)));
        users.add(new User(3, "Jim", LocalDate.now().minusYears(20)));
    }

    public List<User> findAll() {
        return users;
    }

    public User findOne(int id) {
        return users.stream()
                .filter(user -> user.getId().equals(id))
                .findFirst()
                .orElse(null);
    }

    // Save a new user
    public User saveUser(User user) {
        user.setId(++userCount); // Assign a new ID
        users.add(user);
        return user;
    }
}
```

### **ğŸ”¹ What's happening in `saveUser()`?**

- **`++userCount`** â†’ Auto-increments the ID for each new user.
- **`users.add(user)`** â†’ Adds the new user to the list.
- Later, we will replace this **static list** with a **database**.

## âœ… **Enhancements: Returning `201 Created` Response**

Right now, our `POST` request returns **200 OK**, but the proper status code for
resource creation is **201 Created**.

### **ğŸ“Œ Update `createUser()` in `UserResource`:**

```java
@PostMapping
public ResponseEntity<User> createUser(@RequestBody User user) {
    User savedUser = service.saveUser(user);

    // Return 201 Created response with URI
    URI location = ServletUriComponentsBuilder.fromCurrentRequest()
            .path("/{id}")
            .buildAndExpand(savedUser.getId())
            .toUri();

    return ResponseEntity.created(location).body(savedUser);
}
```

1. **`ResponseEntity.created(location)`** â†’ Returns **201 Created**.

### **2ï¸âƒ£ Why can't we use the browser to send `POST` requests?**

- Browsers only support `GET` requests via URL.
- To send `POST` requests, we need **API clients** like:
  - **Postman**
  - **Talend API Tester**
  - **cURL**

---

### **3ï¸âƒ£ Why do we auto-generate IDs instead of accepting them?**

- IDs should be unique and managed by the server.
- In a real application, we use a **database sequence** for ID generation.

---

### **4ï¸âƒ£ Why use `ResponseEntity.created(location)`?**

- It sets the HTTP **201 Created** status.
- It includes the **Location Header** for the newly created resource.
