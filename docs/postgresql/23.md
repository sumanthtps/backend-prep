# Cheatsheet

This PostgreSQL cheatsheet covers **essential SQL commands, performance tuning, security, replication, and advanced features**.  

---

## **1Ô∏è‚É£ Basic PostgreSQL Commands**
| Command | Description |
|---------|-------------|
| `psql -U postgres` | Connect to PostgreSQL as `postgres` user |
| `\l` | List all databases |
| `\c dbname` | Switch to a database |
| `\dt` | List all tables in the current database |
| `\d tablename` | Show table structure (columns, indexes, constraints) |
| `\q` | Exit PostgreSQL shell |

‚úÖ **Creating a Database & User**  
```sql
CREATE DATABASE mydb;
CREATE USER myuser WITH PASSWORD 'mypassword';
GRANT ALL PRIVILEGES ON DATABASE mydb TO myuser;
```

‚úÖ **Dropping a Database & User**  
```sql
DROP DATABASE mydb;
DROP USER myuser;
```

## **2Ô∏è‚É£ Table Management**
‚úÖ **Create a Table**  
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    email TEXT UNIQUE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

‚úÖ **Modify a Table**  
```sql
ALTER TABLE users ADD COLUMN age INT;
ALTER TABLE users DROP COLUMN age;
ALTER TABLE users RENAME COLUMN name TO full_name;
```

‚úÖ **Delete a Table**  
```sql
DROP TABLE users;
```

## **3Ô∏è‚É£ CRUD Operations**
‚úÖ **Insert Data**  
```sql

```

‚úÖ **Select Data**  

‚úÖ **Update Data**  
```sql

```

‚úÖ **Delete Data**  
```sql

```

## **4Ô∏è‚É£ Indexing for Performance**
‚úÖ **Create an Index**  
```sql

```

‚úÖ **Check Query Plan (Index Usage)**  
```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@example.com';
```

‚úÖ **Remove an Index**  
```sql
DROP INDEX idx_users_email;
```

## **5Ô∏è‚É£ Joins**
‚úÖ **INNER JOIN** (Only matching rows)  
```sql
SELECT users.name, orders.amount 
FROM users 
INNER JOIN orders ON users.id = orders.user_id;
```

‚úÖ **LEFT JOIN** (All from left table, matching from right)  
```sql
SELECT users.name, orders.amount 
FROM users 
LEFT JOIN orders ON users.id = orders.user_id;
```

‚úÖ **RIGHT JOIN** (All from right table, matching from left)  
```sql
SELECT users.name, orders.amount 
FROM users 
RIGHT JOIN orders ON users.id = orders.user_id;
```

‚úÖ **FULL JOIN** (All rows from both tables)  
```sql
SELECT users.name, orders.amount 
FROM users 
FULL JOIN orders ON users.id = orders.user_id;
```

## **6Ô∏è‚É£ Transactions & Concurrency**
‚úÖ **Using Transactions**  
```sql
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT; -- Use ROLLBACK instead to undo
```

‚úÖ **Savepoints for Partial Rollbacks**  
```sql
SAVEPOINT sp1;
UPDATE accounts SET balance = balance - 50 WHERE id = 1;
ROLLBACK TO sp1;
```

## **7Ô∏è‚É£ Advanced Queries**
‚úÖ **Full-Text Search**  
```sql
SELECT * FROM articles WHERE to_tsvector(content) @@ to_tsquery('postgres');
```

‚úÖ **JSONB Querying**  
```sql
SELECT details->>'name' FROM users;
SELECT * FROM users WHERE details @> '{"city": "New York"}';
```

‚úÖ **Partitioning (For Large Tables)**  
```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    order_date DATE NOT NULL
) PARTITION BY RANGE (order_date);
```

## **8Ô∏è‚É£ User Management & Security**
‚úÖ **Creating a User with Limited Access**  
```sql
CREATE USER readonly_user WITH PASSWORD 'readonly123';
GRANT CONNECT ON DATABASE mydb TO readonly_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly_user;
```

‚úÖ **Row-Level Security (RLS)**  
```sql
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
CREATE POLICY user_policy ON users USING (id = current_user_id());
```

‚úÖ **Checking Active Connections**  
```sql
SELECT * FROM pg_stat_activity;
```

‚úÖ **Killing a Query**  
```sql
SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE state = 'active';
```

## **9Ô∏è‚É£ Replication & High Availability**
‚úÖ **Enable Logical Replication (`postgresql.conf`)**  
```conf
wal_level = logical
max_replication_slots = 5
```

‚úÖ **Create a Replication Slot**  
```sql
SELECT * FROM pg_create_logical_replication_slot('my_slot', 'test_decoding');
```

‚úÖ **Using `pg_basebackup` for Physical Backup**  
```sh
pg_basebackup -U postgres -D /backup/ -Ft -z -P
```

‚úÖ **Point-in-Time Recovery (PITR)**  
```conf
archive_mode = on
archive_command = 'cp %p /var/lib/postgresql/wal_archive/%f'
recovery_target_time = '2025-03-01 10:30:00'
```

## **üîü Query Performance Optimization**
‚úÖ **Analyze Query Execution Plan**  
```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@example.com';
```

‚úÖ **Vacuum to Remove Dead Tuples**  
```sql
VACUUM ANALYZE;
```

‚úÖ **Parallel Query Execution (Faster Aggregations)**  
```sql
SET max_parallel_workers_per_gather = 4;
```

‚úÖ **Connection Pooling with PgBouncer (`pgbouncer.ini`)**  
```ini
[pgbouncer]
listen_port = 6432
max_client_conn = 500
pool_mode = transaction
```

‚úÖ **Caching Query Results with Redis (Python Example)**  
```python
import redis
r = redis.Redis(host='localhost', port=6379)
result = r.get("query_result") or db.query("SELECT * FROM users")
r.set("query_result", result, ex=60)
```
