# Full-Text Search (FTS)
PostgreSQL **Full-Text Search (FTS)** allows efficient **searching and ranking of text data**. Unlike `LIKE` queries, FTS understands **word meanings, language-specific processing, and ranking results**.  

---

## **1Ô∏è‚É£ Why Use Full-Text Search Instead of `LIKE`?**  
‚úî **`LIKE '%word%'` is slow** because it scans the entire text.  
‚úî Full-text search **tokenizes and indexes words**, making searches **faster and more relevant**.  
‚úî Supports **ranking** of results based on **relevance**.  

‚úÖ **Bad (`LIKE` - Slow, No Ranking)**  
```sql
SELECT * FROM articles WHERE content LIKE '%database%';
```
‚ùå Slow for large text fields.  

‚úÖ **Good (FTS - Fast, Ranked)**  
```sql
SELECT * FROM articles WHERE to_tsvector('english', content) @@ to_tsquery('database');
```
‚úî **Tokenizes & ranks text efficiently.**  

---

## **2Ô∏è‚É£ Understanding `tsvector` & `tsquery`**
| Term       | Description |
|------------|-------------|
| **`tsvector`** | Converts text into a **searchable format** (normalized words). |
| **`tsquery`** | Represents a **search query** (processed input words). |
| **`@@` Operator** | Checks if `tsvector` **matches** a `tsquery`. |

‚úÖ **Example:**  
```sql
SELECT to_tsvector('english', 'PostgreSQL is an advanced database') 
@@ to_tsquery('database');
```
‚úî Returns `TRUE` because `"database"` is found!  

---

## **3Ô∏è‚É£ Creating an FTS-Optimized Table**
‚úÖ **Step 1: Create a Table**  
```sql
CREATE TABLE articles (
    id SERIAL PRIMARY KEY,
    title TEXT,
    content TEXT,
    search_vector tsvector GENERATED ALWAYS AS 
    (to_tsvector('english', title || ' ' || content)) STORED
);
```
üöÄ **The `search_vector` column is automatically updated with indexed text!**  

‚úÖ **Step 2: Index the `search_vector` for Fast Searches**  
```sql
CREATE INDEX idx_articles_search ON articles USING GIN (search_vector);
```
‚úî **GIN Index makes FTS super fast!**  

‚úÖ **Step 3: Query the Table Efficiently**  
```sql
SELECT * FROM articles WHERE search_vector @@ to_tsquery('PostgreSQL & database');
```
‚úî Matches **articles containing both PostgreSQL & database**.  

---

## **4Ô∏è‚É£ Ranking Search Results (`ts_rank`)**
You can **rank results** based on relevance.  

‚úÖ **Example:**  
```sql
SELECT title, ts_rank(search_vector, to_tsquery('PostgreSQL')) AS rank 
FROM articles 
WHERE search_vector @@ to_tsquery('PostgreSQL') 
ORDER BY rank DESC;
```
üöÄ **Results are sorted by relevance!**  

---

## **5Ô∏è‚É£ Advanced Search Features**
‚úÖ **Phrase Search:**  
```sql
SELECT * FROM articles WHERE search_vector @@ phraseto_tsquery('PostgreSQL database');
```
‚úî Finds **exact phrase matches**.  

‚úÖ **Highlight Matching Words:**  
```sql
SELECT ts_headline('english', content, to_tsquery('PostgreSQL')) FROM articles;
```
‚úî **Highlights** matched words in search results.  

‚úÖ **Filtering Stop Words (Common Words)**  
```sql
SELECT to_tsvector('english', 'This is a test') @@ to_tsquery('test');
```
‚úî **Ignores** words like "This" and "is".  

---

## **6Ô∏è‚É£ Optimizing Full-Text Search Performance**
‚úî **Always use a GIN Index:**  
```sql
CREATE INDEX idx_articles_search ON articles USING GIN (search_vector);
```
‚úî **Use `phraseto_tsquery()` for exact phrase matching.**  
‚úî **Store `tsvector` as a GENERATED column** to keep it updated.
